## 「何に困っているか」からスタートする

変革手段を探す前に、まず自社の「痛み」を言語化します。
症状を特定することで、本当に必要な変革手段が見えてきます。

## 症状カテゴリ

| カテゴリ | こんな症状がある方へ |
|---------|------------------|
| [A. 意思決定・戦略](/docs/課題マップ/A_意思決定・戦略) | 決まらない、動けない、方向性がブレる |
| [B. 人・組織文化](/docs/課題マップ/B_人・組織文化) | 人が辞める、採れない、変われない |
| [C. 実行力・オペレーション](/docs/課題マップ/C_実行力・オペレーション) | 決めたことが動かない、業務が非効率 |
| [D. 知見・情報](/docs/課題マップ/D_知見・情報) | 市場が読めない、判断材料がない |
| [E. 成長・外部連携](/docs/課題マップ/E_成長・外部連携) | 成長が止まった、自社だけでは限界 |
| [F. 資金・財務](/docs/課題マップ/F_資金・財務) | 資金が足りない、財務の専門性がない |
| [G. データ・AI対応](/docs/課題マップ/G_データ・AI対応) | AIを活用できていない、データが資産になっていない |

## 使い方

1. 上のカテゴリから「最も刺さる症状」を選ぶ
2. 症状ページで「真因の仮説」を確認する
3. 紐づいた変革手段を確認する

## 課題は単独では起きていない——悪循環ループを理解する

組織の課題は孤立して存在しない。複数の課題が互いを強化し合い、「変われない構造」を作り出している。

### 悪循環ループ①：変革できない組織の自己強化

```
既存業務で手一杯（C-6）
  ↓
変革に着手する時間・余白がない
  ↓
課題が言語化・構造化されない（A-6）
  ↓
何を変えるべきか決まらない（A-1）
  ↓
実行されず元に戻る（C-1）
  ↓
また既存業務で手一杯（C-6）へ……
```

**打開策：** 業務効率化（AI/ツール）で時間を作るか、外部のPMO・副業人材が変革業務を担い、内部の余白を作る。

### 悪循環ループ②：人材流出の自己強化

```
社員の意欲・スキルが見えていない（B-8）
  ↓
活躍の場が与えられない → 静かな退職
  ↓
一部の人に業務が集中（C-7）
  ↓
キーパーソンが過負荷 → 離職（B-1）
  ↓
残った人にさらに集中 → また（C-7）へ……
```

**打開策：** タレントマネジメントで意欲・スキルを可視化し、プロジェクト型組織設計で活躍の場を作る。

### 悪循環ループ③：変革抵抗の自己強化

```
変革への抵抗が強い（B-4）
  ↓
プロジェクトが止まる・形骸化する（C-1）
  ↓
「やっぱり変革は無理」という証拠が増える
  ↓
さらに抵抗が強くなる（B-4）へ……
```

**打開策：** 越境学習で外の成功事例を体験させる、または外部PMO・変革経験者が「動いている状態」を作り、小さな成功体験を積む。
