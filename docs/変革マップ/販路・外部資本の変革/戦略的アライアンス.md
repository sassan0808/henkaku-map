## 概要

他社との業務・資本提携により、自社単独では実現できない事業シナジーを生む手段。
販路・技術・ブランドの相互補完が主な目的。

## アライアンスの類型

| 類型 | 内容 | 例 |
|------|------|-----|
| **販路提携** | 相手の顧客・チャネルに自社製品を乗せる | OEM、代理店契約 |
| **技術提携** | 互いの技術・ノウハウを共有する | 共同開発、ライセンス |
| **コマーシャル提携** | 共同マーケティング・ブランド活用 | 共同キャンペーン、クロスセル |
| **資本提携** | 出資を通じた関係強化 | 少数出資、合弁会社設立 |

## アライアンスが機能しなくなる原因

提携を組んだが成果が出ない、温度差がある、というケースは非常に多い。

- **目的・期待値の合意が曖昧なまま始まった** — 「何のためのアライアンスか」が双方で違う
- **推進オーナーが不在** — 契約後に担当者が誰もいない
- **Win-Winになっていない** — 片方だけが得をする設計になっている
- **最初の成果が出るまでの道筋がない** — 大きなビジョンだけ掲げて、目先の小さな成功がない

## 成功するアライアンス設計の原則

1. **「なぜ自社と組むとメリットがあるか」を相手視点で言語化する** — 相手のメリットが明確でないと機能しない
2. **小さな成果から始める** — 最初から大型案件を狙わず、小さな成功体験を積んで関係を育てる
3. **専任の推進担当を置く** — 「担当者不在の提携」は機能しない
4. **定期レビューを設計する** — 半年・1年ごとに成果を確認し、続けるか変えるかを判断する

## こんな課題に効く

- 新市場・新領域への参入で、既にそこにいる企業の力を借りたい
- 自社にない技術・ノウハウを持つ企業と組むことで補完したい
- 自社単独では取れない大型案件・大企業との取引を狙いたい
