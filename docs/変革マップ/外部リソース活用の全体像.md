## なぜ外部リソースの整理が必要か

「外部の力を借りる」という選択肢は一つではありません。コンサルファームに頼む、顧問を入れる、副業人材を活用する、BPOに出す——それぞれに明確な役割の違いがあります。

これを整理せずに「とりあえず外部に頼もう」で動くと、**コストに見合った成果が出ない**か、**社内に何も残らない**かのどちらかになりやすいです。

---

## 外部リソース活用の比較マトリクス

| 手段 | コスト感 | 役割・期待値 | 社内への知見移転 | 向いている場面 |
|------|---------|-----------|--------------|-------------|
| **BPO / RPO** | 中〜高（月額固定） | 定型業務・採用実務の「巻き取り」 | しない（丸投げ） | 既存業務を切り出し、変革の余白を強制的に作る |
| **コンサルティングファーム** | 極めて高（プロジェクト費） | 戦略策定・大規模プロジェクトの主導 | しにくい（ブラックボックス化しやすい） | 絶対に失敗できない全社レベルの変革 |
| **外部プロ人材**（副業・フリーランス） | 低〜中（時間単価） | 戦略の解像度を持ちながら泥臭い実務をリード | する（並走・OJT） | 新規事業・DXなど動きながら正解を探す変革 |
| **顧問・アドバイザリー** | 中（月額顧問料） | 意思決定の壁打ち・人脈の提供 | しない（経験は本人に帰属） | 経営陣の判断力強化・信用補完 |
| **スポットコンサル** | 低（時間単価） | 特定領域の知見を単発で獲得 | する（知識の移転） | 方向性の検証・初動の仮説出し |

---

## 「知見移転するかどうか」が最重要の選択軸

外部リソースの活用で最も失敗しやすいのが、**「終わったら何も残っていない」**問題です。

```
知見移転しない外部活用（ブラックボックス型）
  外部が優秀 → 成果が出る → 契約終了
  → 社内に何も残らない → また外部に頼る → コストが永続する

知見移転する外部活用（OJT型）
  外部が優秀 → 成果が出る + 社内メンバーが並走して学ぶ
  → 社内にケイパビリティが残る → 外部依存が段階的に減る
```

**使う目的によって選ぶべき手段が変わります：**

- **余白を作りたい（定型業務を切り出す）** → BPO/RPO
- **絶対に失敗できない大型変革** → コンサルティングファーム
- **動きながら組織のケイパビリティも上げたい** → 外部プロ人材（副業・フリーランス）
- **経営者の判断を支えたい** → 顧問・アドバイザリー
- **まず知見を得て方向性を決めたい** → スポットコンサル

---

## 組み合わせ設計の考え方

複数の手段を組み合わせるのが実際の変革では一般的です。

**よくある組み合わせパターン：**

```
① 方向性を決める段階
  スポットコンサル → 方向性の仮説を短時間で検証

② 動き出す段階
  外部プロ人材（副業）→ 実務の推進役として並走
  + 顧問 → 意思決定の壁打ち・経営層への説得

③ スケールする段階
  BPO → 増えた定型業務を外出しして社内余白を維持
  コンサルファーム → 全社展開・大型システム刷新など、規模が必要になった局面で

④ 自走する段階
  外部依存を段階的に縮小 → 社内で回る状態へ
```
