## 概要

顧客が契約後に「成果・価値」を実感できるよう、継続的に支援する体制を構築する手段です。
解約を防ぎ、LTV（顧客生涯価値）を最大化することが目的です。

---

## なぜカスタマーサクセスが重要か

SaaSやサブスクリプションモデルの普及により、**「売って終わり」の営業では事業が成り立たない**時代になっています。

```
旧モデル（売り切り・顧客管理）
  受注 → サポート対応（問い合わせがあれば応答） → 更新 or 解約
  → 解約しても「次を売れば良い」で成立した

新モデル（カスタマーサクセス）
  受注 → オンボーディング → 価値実感の確認 → 活用支援 → 継続・拡張
  → 解約=収益の喪失。1件の継続が新規獲得より安い
```

**LTVの観点：** 新規顧客の獲得コスト（CAC）は継続顧客の維持コストの5〜7倍かかると言われています。解約を1%下げるほうが、新規を1%増やすより経営インパクトが大きいです。

---

## カスタマーサクセスの主な手段

### 1. オンボーディング設計

契約直後の「最初の価値実感」を設計します。

- 導入ステップを明確化し、顧客が迷わないようにする
- 「最初の成功体験（First Value）」をどこに設定するかを決める
- チェックリスト・ウォークスルー・担当者との初回ミーティングの設計

**なぜ重要か：** 最初の数週間〜1ヶ月で価値を感じられなかった顧客は、その後も使い続けません。解約リスクは契約直後から始まっています。

### 2. ヘルススコアの設計と管理

顧客の「健全度」を数値化し、解約リスクを早期検知します。

**ヘルススコアの構成例：**
- ログイン頻度・利用率
- 主要機能の利用状況
- サポート問い合わせの頻度・内容
- NPS（推薦意向スコア）
- 支払い状況

スコアが下がった顧客に先手でアプローチすることで、解約を未然に防ぎます。

### 3. ハイタッチ・ロータッチ・テックタッチの設計

すべての顧客に同じコストをかけるのではなく、LTVに応じて対応を分けます。

| タイプ | 対象 | 対応 |
|------|------|------|
| ハイタッチ | 大口・戦略顧客 | 専任担当者がついて伴走 |
| ロータッチ | 中規模顧客 | 定期ウェビナー・グループ支援 |
| テックタッチ | スモール・多数顧客 | メール自動配信・動画・FAQで自己解決 |

### 4. アップセル・クロスセルの設計

CSは「守り（解約防止）」だけでなく「攻め（拡張）」の役割も持ちます。

- 利用が定着した顧客への上位プランの提案
- 他の製品・サービスの紹介
- 成功事例のリファレンス・口コミ依頼

---

## BtoB以外への応用

カスタマーサクセスの考え方は BtoB SaaS だけでなく幅広く使えます。

| 業態 | 応用 |
|------|------|
| 飲食・小売 | 来店後フォロー、ロイヤル顧客プログラム |
| 士業・コンサル | 契約後の支援体制、成果の可視化 |
| 教育・スクール | 受講後の定着支援、卒業生コミュニティ |
| EC | 購入後のフォローアップ、レビュー依頼 |

---

## こんな課題に効く

- 解約・離脱が多い
- 既存顧客からの売上が伸びない
- 顧客対応が問い合わせ対応だけで終わっている
- 顧客がどれだけ活用しているか把握できていない
- リファレンス・口コミが生まれない
