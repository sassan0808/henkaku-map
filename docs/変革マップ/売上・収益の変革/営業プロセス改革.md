## 概要

属人的な「営業スタイル」を脱し、再現性のある営業プロセスとして仕組み化する手段です。
トップ営業の暗黙知を可視化・標準化し、チーム全体の底上げを図ります。

---

## 属人営業の何が問題か

| 属人営業の状態 | 仕組み化された状態 |
|------------|---------------|
| 誰が担当するかで成約率が変わる | プロセスを踏めば一定の成果が出る |
| なぜ受注・失注したか分からない | 商談データから勝ちパターンが見える |
| 担当が抜けると顧客関係ごと消える | 顧客情報・経緯がチームで共有される |
| 採用・育成に時間がかかりすぎる | 再現可能なトレーニングができる |

---

## 営業プロセス改革の主な手段

### 1. SFA（営業支援システム）の導入・活用

SFA（Salesforce, HubSpot, Sansan等）で商談の全ステップをデータ化します。

**整備すべきデータ：**
- リードの発生源（どこから来たか）
- 商談ステージ（初回接触→提案→クロージング）
- 失注理由の分類
- 担当者別・製品別・時期別の転換率

**よくある失敗：** ツールを入れても「入力がされない」。入力コストを下げる設計と、マネージャーが会議でデータを使う習慣作りがセットです。

### 2. 商談プロセスの標準化

```
リード獲得
  ↓
初回ヒアリング（課題の確認・BANT情報の収集）
  ↓
提案書作成（課題→解決策→ROIの構造）
  ↓
クロージング（懸念点の解消・意思決定者の巻き込み）
  ↓
契約・オンボーディング（CSへの引き継ぎ）
```

**BANTとは：** Budget（予算）、Authority（決裁権）、Need（必要性）、Timeline（時期）の略です。初回ヒアリングで把握すべき基本情報です。

### 3. 営業ナレッジの蓄積と共有

- 勝ちパターン・負けパターンの言語化
- 反論処理集（よくある懸念への返し方）
- 業界別・規模別の提案テンプレート
- ロールプレイ・商談録音の活用

### 4. パイプライン管理

毎週のパイプラインレビューで「今月・来月の受注予測」を可視化します。
感覚ではなくデータで「何が足りないか」を判断できる状態を作ります。

## AI時代の営業プロセス

AIを活用することで、営業の高度化が一気に加速します。

| 活用シーン | AIの役割 |
|----------|---------|
| 提案書作成 | ヒアリング内容→提案書の下書き自動生成 |
| 失注分析 | 商談データから失注パターンを自動抽出 |
| 優先度付け | 受注確度スコアリング（リードのランク付け） |
| 顧客調査 | 商談前の企業リサーチを自動化 |

---

## こんな課題に効く

- 営業成績が担当者によってバラバラ
- なぜ受注・失注したか分からない
- 新人が育つまでに時間がかかりすぎる
- 営業が何件商談しているか把握できていない
- マネージャーが現場の状況を「感覚」で管理している
