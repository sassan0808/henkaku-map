## このカテゴリの目的

組織変革の最終的な目的は、事業の成長だ。しかし変革の議論になると「組織を整える」「効率化する」に終始し、**売上を生み出す核心部分（営業・マーケティング・CS）の変革**が抜け落ちることが多い。

このカテゴリでは、収益を生み出す3つの機能を変革する手段を扱う。

```
マーケティング → 営業 → カスタマーサクセス
   （集客）      （受注）     （継続・拡大）
        ↑─────────────────────────────────┘
               RevOpsでデータをつなぐ
```

---

## 手段一覧

| 手段 | 一言で言うと | 優先度が高いケース |
|------|-----------|----------------|
| [営業プロセス改革](/docs/変革マップ/売上・収益の変革/営業プロセス改革) | 属人的な営業を仕組みに変える | 営業成績にばらつきがある、商談管理ができていない |
| [マーケティング変革](/docs/変革マップ/売上・収益の変革/マーケティング変革) | 見込み客を仕組みで集める | 紹介・既存依存、広告だけに頼っている |
| [カスタマーサクセス](/docs/変革マップ/売上・収益の変革/カスタマーサクセス) | 顧客が成功する体験を設計する | 解約・離脱が多い、LTVが低い |
| [RevOps導入](/docs/変革マップ/売上・収益の変革/RevOps導入) | マーケ・営業・CSをデータでつなぐ | どこで失注・解約しているか見えない |
