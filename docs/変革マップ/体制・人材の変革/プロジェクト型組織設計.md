## 概要

既存の組織図・役職体系から独立したプロジェクトベースの組織単位を設計・運営する手段。
変革業務・新規事業・横断課題に対し、**役職や年次ではなくスキルと意欲を基準に人材を編制する**。

これからの時代、変革を推進するには「既存組織図の枠内でやる」発想から脱することが必要。
プロジェクトベースのアジャイル組織は、硬直した縦割り構造の「抜け穴」として機能する。

## 主な組織設計パターン

| パターン | 特徴 | 向いているケース |
|---------|------|---------------|
| **タスクフォース（TF）** | 特定課題に対して期間限定で編成。終わったら解散 | 変革プロジェクト・緊急課題対応 |
| **スクワッド** | 小規模の自律チームが継続運営。アジャイル開発で一般的 | 新規事業・プロダクト開発 |
| **CoE（センター・オブ・エクセレンス）** | 特定専門領域の横断チーム。各部門を支援 | DX推進・データ活用・AI推進 |
| **兼務プロジェクト制** | 既存業務を持ちながら、一定割合をプロジェクトに割く | リソースが限られる中小企業 |

## 設計のポイント

### 1. 評価制度との連動が最重要
プロジェクト参加が「プラス評価」にならないと誰も動かない。
- プロジェクト貢献を評価項目に組み込む
- 抜擢された人材のキャリアパスを明示する
- 「既存業務の免除」をセットで設計する

### 2. アサイン基準を役職・年次から変える
```
旧：役職者 or 年次が上の人が自動的にリーダー
新：スキル・意欲・適性でアサイン → 抜擢人事
```
意欲のある若手・専門スキルを持つ人材が活躍できる仕組みが必要。

### 3. 稼働の実態を把握する
「みんな忙しそう」という空気感で止まらないために、
実際の稼働率・業務量の可視化が先決。本当に余裕がないのか、
それとも既存業務が非効率なだけかを切り分ける。

### 4. 外部人材をプロジェクトに組み込む
社内人材だけで閉じない。副業・フリーランスの外部人材を
最初からチームに入れることで、社内の「常識」を壊すきっかけになる。

## 「外部を軸に、内部が学ぶ」モデル

変革プロジェクトを設計するとき、暗黙の前提として「社内の上位レイヤーがリードすべき」という発想が働きやすい。しかし、既存組織の管理職や役職者が、変革業務に必要な専門性や指向性を持っているとは限らない。特に、既存事業とは異なるドメインへの新規事業や、DX・AI・新市場開拓といった変革テーマでは、この前提が大きな障壁になる。

こうした場合に有効なのが、**「外部人材を軸に、社内人材が学ぶ」**という設計モデルだ。

```
旧モデル（役職ベース）
  組織の上位者が自動的にプロジェクトリーダー
  → 専門性・意欲がなくても責任者になる
  → プロジェクトが形骸化、または推進力を失う

新モデル（専門性×意欲ベース）
  ┌──────────────────────────────────┐
  │ プロジェクト軸：外部人材           │
  │  └ 専門知見・実行経験を持つ        │
  │    副業・フリーランス・顧問等       │
  ├──────────────────────────────────┤
  │ 推進担当：社内の意欲ある人材（抜擢）│
  │  └ 役職・年次ではなく               │
  │    スキルと意欲で選ぶ              │
  └──────────────────────────────────┘
```

### このモデルの3つのメリット

**① 変革スピードの確保**
外部人材は最初から「やり方」を知っている。社内が試行錯誤する時間を省き、初動から質の高いプロジェクト運営が可能になる。

**② 社内への知見移転**
外部人材と並走する中で、抜擢された社内メンバーは実務を通じて専門性を習得する。研修ではなく「現場で学ぶ」最速の育成モデルになる。

**③ コストの柔軟なコントロール**
社内人材が十分なスキルを吸収したタイミングで、外部人材への依存を段階的に減らす。「必要な期間だけ使う」ことで、固定費を増やさずに組織のケイパビリティを底上げできる。

### 設計の注意点

- 外部人材に「丸投げ」するのではなく、社内メンバーが必ず並走する体制を作る（知見移転が目的）
- 外部人材が抜けたあとの「引き継ぎ計画」を最初に設計しておく
- 外部人材の知見が属人化しないよう、ドキュメント化・仕組み化をプロジェクトの成果物に含める

## こんな課題に効く

- 新規プロジェクトを立ち上げようとすると「誰がやるの？」で止まる
- 変革を推進したいが、既存組織の硬直構造が邪魔をしている
- 部分最適・サイロ化が強く、全社視点で動けない
- 意欲・スキルのある人材が既存業務に縛られて動けない
- 評価制度が変革参加の障壁になっている
