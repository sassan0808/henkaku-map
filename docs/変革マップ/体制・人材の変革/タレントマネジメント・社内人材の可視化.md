## 概要

社員のスキル・経験・意欲・キャリア志向を構造化して把握し、
**適材適所の配置・抜擢・育成**を組織的に実現する手段。

「やる気がない人が多い」ではなく、「やる気を削ぐ構造になっている」という視点から、
人材データの整備と活用で組織の潜在能力を引き出す。

## なぜ今これが重要か

- **静かな退職（Quiet Quitting）** の広がり：やる気を失った社員が最低限の仕事しかしなくなる現象。問題は「人のやる気」ではなく「やる気を削ぐ組織構造」にある
- **タレントロス**：社内で活躍できずに転職していった人材のうち、多くは「見つけてもらえなかった」だけ
- **HRサイロ**：人材データが採用・評価・育成でバラバラに管理され、全体像が見えない
- **AI時代の人材配置**：スキルの陳腐化が早く、固定的な職務定義では対応できない

## 主な手段

### 1. スキル・意欲データの構造化

```
収集項目（例）
- 現在のスキルセット（専門スキル・汎用スキル）
- 過去の経験・プロジェクト実績
- やりたいこと・挑戦したいこと
- 苦手・やりたくないこと
- キャリア志向（管理職希望 / スペシャリスト志向）
- 副業・社外活動での経験
```

### 2. 定期的な1on1・キャリア面談の設計

- 「業務の進捗確認」ではなく「本音を引き出す場」として設計する
- 形式：月次1on1（上司）＋半期キャリア面談（HR or 第三者）
- マネージャーに1on1スキルトレーニングをセットで行う

### 3. 社内公募・プロジェクト参加制度

- 新規プロジェクトのメンバー募集を社内オープンに
- 立候補制・推薦制を組み合わせて「見つけてもらえる」仕組みを作る
- 参加実績を評価に反映する

### 4. タレントマネジメントシステム（TMS）の導入

| ツール種別 | 目的 | 例 |
|-----------|------|-----|
| HRISと連携したスキル台帳 | 一元管理 | SmartHR, Workday |
| 社内SNS・プロフィール共有 | 相互発見 | Notion, Confluence |
| ピープルアナリティクス | 離職予測・配置最適化 | カオナビ, タレントパレット |

## 設計のポイント

### やる気を「測る」のではなく「削がない構造」を作る

```
問い方が間違っている例：「なぜ社員のモチベーションが低いのか？」
正しい問い：「組織のどの構造がやる気を削いでいるか？」

チェックすべき構造的問題：
- 裁量がない（全て上司の承認が必要）
- 成長機会がない（同じ業務を何年も繰り返す）
- 評価が不透明（何をすれば認められるか分からない）
- 挑戦を責める文化（失敗したら詰められる）
- 意欲を示す場がない（言っても無駄という諦め）
```

### データは集めるだけでなく「使う」仕組みが必要

- 収集したスキル・意欲データを人事異動・プロジェクトアサインに実際に使う
- 「データがあるのに使われていない」状態が一番信頼を失う

## こんな課題に効く

- 採用コストをかけているのに定着率が低い
- 優秀な人材が「社内で活躍できなかった」という理由で転職している
- 誰が何のスキルを持っているか、HRも把握できていない
- プロジェクトのアサインが「いつもの顔触れ」になっている
- 静かな退職・エンゲージメント低下が深刻化している
