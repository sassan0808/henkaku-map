## 概要

テクノロジーで業務プロセスを自動化・高度化する手段。
人手に依存していた作業をシステムに置き換え、スピードと精度を上げる。

## ツール導入が失敗するパターン

「ツールを入れたが使われなかった」は非常に多い。原因はほぼ共通している。

- **現場ニーズとのズレ** — ITや経営が選んだが、現場が「使いにくい」と戻ってしまう
- **運用設計の欠如** — 誰がどう使うか、ルールや習慣が設計されていない
- **研修・トレーニング不足** — 使い方を教えないまま導入した
- **既存業務フローを変えずに入れた** — 古いフローにツールを乗せても効果が出ない
- **ゴールが「導入」になっている** — ツールを入れることが目的化し、使われているかを追わない

## ツール選定の基準

| 観点 | チェックポイント |
|------|--------------|
| 現場フィット | 実際に使う人が使いやすいか。PoC（試験導入）で確認したか |
| 運用コスト | 管理・メンテナンスの負荷は許容範囲か |
| 拡張性 | 将来の業務変化に対応できるか。データを他ツールと連携できるか |
| サポート | 問題発生時のサポート体制はあるか |
| 導入実績 | 同業・同規模の企業での導入実績があるか |

## 主なツールカテゴリ

| カテゴリ | 目的 | 代表例 |
|---------|------|--------|
| CRM | 顧客管理・営業支援 | Salesforce, HubSpot |
| MA | マーケティング自動化 | Marketo, Braze |
| HR/タレント | 人事・採用・評価 | SmartHR, カオナビ |
| PM | プロジェクト管理 | Notion, Asana, ClickUp |
| BI・可視化 | データダッシュボード | Looker, PowerBI |
| コミュニケーション | 社内連絡・ナレッジ共有 | Slack, Confluence |
| 会計・財務 | 経理・CFO支援 | freee, マネーフォワード |
| AI活用 | 業務自動化・生成AI | ChatGPT API, Dify |

## 定着させるための設計

1. **スモールスタート** — 全社一斉展開ではなく、1部門・1業務から始める
2. **社内チャンピオンを作る** — 各部門に「使いこなしている人」を育てる
3. **使用状況を可視化する** — ログイン率・利用率を追って放置を早期発見する
4. **業務フローとセットで変える** — ツール導入と業務改革（BPR）は同時設計が理想
