## 概要

外部のプロフェッショナルがプロジェクト管理・実務を代行・伴走する手段。
「わかってはいるが動けない」組織の実行力を補完する。

「決めたことが動かない」原因のほとんどは、意欲や知識の不足ではなく、**管理する仕組みと責任主体の不在**にある。PMOはそこを外部から埋める。

## PMOが担う役割

| 機能 | 内容 |
|------|------|
| プロジェクト管理 | タスク・スケジュール・進捗の一元管理 |
| KPI設計・追跡 | 何をもって成功とするかを定義し、数字で追いかける |
| リスク管理 | 問題が起きる前に兆候を察知して対処する |
| ステークホルダー調整 | 関係部門・経営層との合意形成・報告設計 |
| 意思決定の促進 | 判断が止まっている箇所を整理し、決める場を設計する |
| 社内人材の育成 | 並走しながら、社内に実行管理の能力を移転する |

## 外部PMOを入れる価値

社内でPMOを設置しようとしても、「既存業務を抱えたまま兼務」になることが多く、機能しない。外部PMOには次の強みがある。

- **政治から独立している** — 社内力学に影響されず、事実ベースで動ける
- **即戦力** — プロジェクト管理の型・ツールをそのまま持ち込める
- **コミット量を調整できる** — 週次稼働からフル常駐まで、フェーズに合わせて変えられる
- **"外圧"として機能する** — 社内への変革推進に「外部がそう言っている」という文脈が使える

## 向いている場面

- 大型変革プロジェクト（基幹システム導入、BPR、組織再編）
- 複数部門をまたぐ横断プロジェクトで、誰も主体になれない
- プロジェクトが炎上・停滞しており、立て直しが必要
- 経営と現場の乖離が激しく、推進力が生まれない

## 伴走フェーズの設計

PMOを入れて終わりにしないために、フェーズを設計することが重要。

```
フェーズ1：PMOが主導（3〜6ヶ月）
  外部が管理・推進の中心を担いながら、社内の動き方を設計する

フェーズ2：共同運営（3〜6ヶ月）
  社内担当と並走。徐々に社内主導に移行しながら、ノウハウを移転する

フェーズ3：PMO撤退・社内定着
  社内に実行管理の仕組みと人材が残る状態で終了する
```

## よくある失敗パターン

- **「お目付役」として現場が反発する** — PMOは管理ではなく「動かすための支援」という文脈が重要
- **報告整備だけで終わる** — 報告書を作ることがゴールではなく、判断と実行が動くことがゴール
- **社内への知識移転を設計しない** — PMOが抜けた後に元に戻る
