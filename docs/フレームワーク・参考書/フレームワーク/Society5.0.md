## 一言で言うと

> Society 4.0（情報社会）では「データはあるが分断されている」。Society 5.0はそのデータを統合し、AI・サイバー空間と現実を融合させることで、新しい価値を生み出す社会だ。

---

## Societyの進化段階

日本政府（内閣府）が定義した、人類社会の発展段階の整理。

| ステージ | 名称 | 特徴 |
|---------|------|------|
| Society 1.0 | 狩猟社会 | 自然に依存。食料・安全の確保が中心 |
| Society 2.0 | 農耕社会 | 自然を活用。定住・集団の形成 |
| Society 3.0 | 工業社会 | 機械で大量生産。資本・労働が価値の源泉 |
| Society 4.0 | 情報社会 | コンピュータ・インターネットで情報流通が加速 |
| **Society 5.0** | **超スマート社会** | **サイバーと現実を融合。データが人・物・社会をつなぐ** |

---

## Society 4.0の限界：データはあるのに使えない

インターネットが普及し、膨大なデータが生まれた。しかし Society 4.0 には根本的な問題がある。

```
Society 4.0 の構造的問題

  医療データ    │ 行政データ    │ 企業データ    │ 個人データ
  ──────────── │ ──────────── │ ──────────── │ ────────────
  病院ごとに    │ 省庁ごとに    │ 部門ごとに    │ サービスごとに
  サイロ化      │ サイロ化      │ サイロ化      │ サイロ化
  ↓
  データはあるのに横断活用ができない
  → 意思決定が遅い・非効率・格差が縮まらない
```

**これは企業の中でも同じ構造で起きている。**
営業データ・マーケデータ・製造データ・人事データが縦割り部門に分散し、統合されていない。

---

## Society 5.0が目指すもの

**サイバー空間（デジタル）と現実空間（フィジカル）を高度に融合させる。**

```
Society 5.0 の構造

  フィジカル空間（現実）
    人・物・インフラ・自然
    ↓ センサー・IoT・行動データ
  サイバー空間（デジタル）
    データが統合・分析・AI処理
    ↓ フィードバック・最適解の提示
  フィジカル空間（現実）
    人が最適な行動・意思決定を取る
```

この「現実→デジタル→現実」の循環が高速・精密になるほど、社会・組織・個人の能力が拡張される。

### Society 5.0が解決しようとする課題

- 少子高齢化・労働力不足
- 地域間格差・医療格差
- エネルギー・食料の最適配分
- 行政サービスの効率化
- 企業の生産性向上とウェルビーイングの両立

## 企業にとっての意味：DXの「本当のゴール」

Society 5.0 の文脈で捉えると、企業の DX は「ツールを導入すること」ではない。

**DXの本質 = データを統合し、サイバー・フィジカルの循環を自社内・自社外で作ること**

```
DXの3段階（Society 5.0 の文脈）

  Lv1 デジタイゼーション：紙→データ化（アナログをデジタルに置き換える）
  Lv2 デジタライゼーション：業務プロセスをデジタルで効率化
  Lv3 デジタルトランスフォーメーション：データが統合され、ビジネスモデルと社会の仕組みが変わる
                                          ← ここが Society 5.0 の入り口
```

多くの日本企業は Lv1〜Lv2 で止まっており、「ツールを導入したが、データがバラバラのまま」という状態になっている。

---

## AI時代との接続：AIはデータ統合の先にある

AI の本当の価値は「統合されたデータ」があってはじめて発揮される。

```
AI活用の前提条件

  部門ごとのサイロデータ
    → AIができること：単体業務の自動化（翻訳・要約・コード生成など）
    → AIができないこと：全社最適・事業判断・顧客体験の設計

  統合されたデータ（Society 5.0 的基盤）
    → AIができること：リアルタイム意思決定支援・予測・異常検知・最適化提案
    → ここが「AIレディな組織」の意味
```

**「AIを使いこなせるか」の差は、AIのスキルより「データ統合ができているか」の差だ。**

---

## 日本企業が Society 5.0 に向かうための実践ステップ

### 1. 現状把握：自社のデータはどこにあるか
- 各部門が持つデータの棚卸し
- 重複・欠損・サイロの可視化

### 2. データ統合の設計
- データ基盤（DWH・データレイク）の構築
- BizOps・RevOps の考え方でKPIを統一設計（[BizOps・RevOps](/docs/フレームワーク・参考書/フレームワーク/BizOps-RevOps) 参照）

### 3. AIリテラシーの底上げ
- データを読む・問いを立てる・AIに指示する能力を全社員に
- 特定スキルではなく横断的な汎用スキルとして扱う（[AIリテラシー・活用研修](/docs/変革マップ/データ・AI活用の変革/AIリテラシー・活用研修) 参照）

### 4. サイバー・フィジカル循環の設計
- 現場データ → 分析 → 現場へのフィードバックの仕組みを作る
- 「データが意思決定に使われる文化」を作る

---

## まとめ：Society 5.0 を羅針盤として使う

Society 5.0 は「政府の掲げる理想論」ではなく、**今の変革の方向性を判断する羅針盤**として使える。

> 「この取り組みは Society 5.0 に向かっているか？」
> → データが統合される方向か？
> → サイバー・フィジカルの循環が生まれるか？
> → AI が意思決定に使われるようになるか？

これが「Yes」なら正しい方向。「No（ツール導入だけで終わる）」なら再設計が必要だ。
