## OKRとは何か

**OKR（Objectives and Key Results）** は、インテルのアンドリュー・グローブが開発し、Googleが全社導入したことで世界に広まった目標管理フレームワーク。

```
Objective（目標）：定性的・意欲的な「向かいたい方向」
Key Results（主要な成果）：Objectiveの達成を測る定量的な指標（2〜5個）
```

**例：**
- Objective：「顧客が熱狂するプロダクトを届ける」
- Key Result 1：NPS（顧客推薦度）を30から50に上げる
- Key Result 2：解約率を5%から2%に下げる
- Key Result 3：月次アクティブユーザーを1万から2万に増やす

## MBOとの違い

日本企業で一般的な **MBO（Management by Objectives）** と混同されやすいが、本質的に異なる。

| 観点 | MBO | OKR |
|------|-----|-----|
| 目標の設定者 | 上司が設定（トップダウン） | 本人が設定（ボトムアップ中心） |
| 達成率の扱い | 100%達成が標準 | 60〜70%達成が理想（低すぎると挑戦不足） |
| 評価・給与との連動 | 強く連動する | 原則として連動させない |
| 公開範囲 | 基本的に非公開 | 全社に公開する |
| サイクル | 年次 | 四半期（3ヶ月） |

**最大の違い：OKRは「評価ツール」ではなく「アライメントツール」だ。**
全社・チーム・個人のOKRが公開されることで、「自分の仕事が会社全体のどこに貢献しているか」が見える。

## OKRが解決する組織の問題

### サイロ化の解消

MBOでは各部門が自分のKPIを追うため、部門間の優先度が衝突することが多い。
OKRは全社のObjectiveを起点にカスケード（連鎖）するため、部門間の方向性が揃いやすい。

```
全社OKR：「市場シェアNo.1になる」
  ↓ カスケード
営業チームOKR：「新規顧客100社獲得」
マーケチームOKR：「リード1000件創出」
プロダクトチームOKR：「解約率3%以下」
```

### 現状維持バイアスの打破

MBOの「達成できる目標を立てる」文化では、挑戦的な目標が設定されにくい。
OKRは「**ムーンショット目標**（到底無理に見える高い目標）」を推奨し、60〜70%達成でも成功とみなす。

これにより、失敗を恐れない挑戦の文化が生まれる。

### 自律性の向上

目標を自分で設定し、全社に公開する設計が、「やらされ感」ではなく「自分事」の感覚を生む。
これは「静かな退職」「変革への抵抗」の根本である**自律性の欠如**に直接効く。

## OKR導入の落とし穴

**日本企業での失敗パターン：**

1. **MBOの上書きとして使う** — 達成率100%を求め始め、挑戦的な目標が出なくなる
2. **評価と連動させてしまう** — 給与への影響を恐れ、低い目標を設定するようになる
3. **目標が多すぎる** — Objectiveは3〜5個が上限。多すぎると焦点が散る
4. **四半期で更新しない** — 年次運用になり、OKRの高速サイクルの価値がなくなる
5. **経営層が公開しない** — 「全社公開」が形骸化し、アライメントが機能しない

---

## 実務での使い方

- **変革プロジェクトの目標設定** — 「導入完了」ではなく「業務時間30%削減」のようなアウトカム指標を設定する
- **評価制度改革のファーストステップ** — 全面的な制度改革の前に、特定チームでOKRを試験導入する
- **部門横断プロジェクトのアライメント** — 各部門のOKRを上位OKRに接続することでサイロを解消する
